#redux

Redux简介

Redux是JavaScript状态容器，提供可预测化的状态管理,采用了发布订阅设计模式

store

Redux的createStore方法创建的库，整个应用只有一个store

state

store维护的状态，本质是一个js普通对象

action

行为，本质是js普通对象，它必须有一个type属性，type属性在reducer处理时要用，除了type外你可以携带其他属性数据，提供给reducer，应该尽量减少在action中携带的数据

dispatch

派发action给reducer处理

reducer

处理dispatch派发过来的action，处理后返回的结果store会将其替换成当前state

redux(版本4.0.4)源码解读

createStore.js

getState

读取store的状态树state，返回当前state

subscribe

注册订阅函数，在reducer处理完后将会用订阅函数来通知

dispatch

唯一改变state的方法，在里面使用reducer处理action并将结果替换为currentState,运行订阅函数通知订阅者

replaceReducer

在使用中途替换reducer处理函数

isPlainObject.js

判断值是不是普通js对象
普通对象指的是字面量创建let obj = {}，或者是构造函数Object new出来的 let obj = new Object()
它们都是构造函数Object new出来的实例对象,而Object new出来的实例对象的原型指针__proto__都指向Object的prototype,最终Object.prototype的__proto__指向null，这就是原型链（如下图包括整个原型链的解释，所有的实例对象都指向构造函数的prototype,所有的构造函数的prototype都指向Object.prototype,最终指向null）

发布订阅模式（也叫观察者模式）

它定义对象间的一种一对多的依赖关系当一个对象状态发生改变时，所有依赖它的对象都将得到通知
